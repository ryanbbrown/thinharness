Opinions

the reason it stays small
no_bash

No bash

Business agents don't need a shell. Cut a giant security surface and a whole class of failures stops being possible.

skills

Skills are tools, not auto-discovery

Skills live in directories you point at explicitly. The agent calls them like any other tool. No magic scan, no marketplace.

search

Search is a top priority

Ripgrep exposed as compact grouped results, tuned for documents and business workflows — plus a JSONL variant for structured data.

parallel_llm

Parallel LLM calls, built in

Fan out from inside the harness when a workflow needs reliability beyond a single loop — majority vote, ensembled extraction.

providers

Three providers, no matrix

Small classes for OpenAI, Anthropic, and OpenRouter. Swap a base URL and move on, or fork one — they're tiny.

no_compaction

No compaction

The right answer to "context is getting big" is better task decomposition: shorter runs, separate instances, narrower subagents.
